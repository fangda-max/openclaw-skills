# 系统功能需求说明书 · 文档结构模板

本文件定义了需求说明书的完整章节结构。生成文档时按此结构组织内容，根据项目规模裁剪可选章节。

---

## 章节必选/可选说明

| 章节 | 必选性 | 轻量项目 | 标准项目 | 完整项目 |
|------|--------|----------|----------|----------|
| 一、背景与建设目标 | **必选** | 保留 | 保留 | 保留 |
| 二、范围与边界 | 推荐 | 合并入第一章 | 保留 | 保留 |
| 三、总体业务流程概览 | 推荐 | 合并入第四章各模块 | 保留 | 保留 |
| 四、系统功能 | **必选** | 保留 | 保留 | 保留 |
| 五、非功能需求 | 推荐 | 精简为1节 | 保留 | 保留 |
| 六、系统接口 | 可选 | 省略 | 有集成时保留 | 保留 |
| 七、权限管理 | 可选 | 合并入第四章 | 角色多时保留 | 保留 |

- **必选**：任何规模都必须包含
- **推荐**：标准及以上项目保留，轻量项目可合并或精简
- **可选**：根据项目实际需要决定

## 各章节详细模板

### 一、背景与建设目标

#### 1.1 背景

写作要求：
- 第一段：描述业务领域、当前业务流程中涉及本系统的关键环节
- 第二段：描述当前方式的具体痛点（3-5个，用列表展开）
- 第三段：总结性过渡句，引出建设必要性

格式示例：
```
当前，在XXX业务过程中，XXX等多个关键环节涉及XXX。
这些XXX长期以来完全依赖人工方式，暴露出显著弊端：
- 效率低下：XXX
- 标准不一：XXX
- 风险积聚：XXX
基于上述背景，传统模式已无法满足XXX的要求，亟需通过技术手段进行变革。
```

#### 1.2 建设目标

写作要求：
- 总述句：一句话概括系统定位
- 分条列出3-5个核心目标，每条包含"目标名称"和"具体说明"
- 结尾总结句：描述实现目标后的整体效果

格式示例：
```
本需求旨在XXX范围内，建设一个XXX系统，实现从XXX向XXX的转型。

核心建设目标包括以下N个方面：
1. 目标一名称：具体说明...
2. 目标二名称：具体说明...
3. 目标三名称：具体说明...

通过达成以上目标，本需求将构建起一个"XXX → XXX → XXX"的闭环。
```

#### 1.3 术语与定义（若有行业术语或内部概念）

| 术语 | 定义 |
|------|------|
| XXX | XXX的含义 |

### 二、范围与边界

#### 2.1 建设的内容范围

写作要求：
- 将范围分为2-3个层次描述（如基础层、核心层、扩展层）
- 每层用"层名 – 一句话定位"作标题，下面列出具体内容

格式示例：
```
第一层：基础数据层 – XXX
- 要点1
- 要点2

第二层：核心业务功能层 – XXX
- 要点1
- 要点2

第三层：支撑与扩展层 – XXX
- 要点1
- 要点2
```

#### 2.2 系统建设的明确边界

写作要求：
- 分两部分：**包含在范围内** 和 **不包含在范围内**
- 每部分用列表逐条说明

```
包含在范围内的业务与功能：
- XXX
- XXX

不包含在范围内的业务与能力：
- XXX：原因说明
- XXX：原因说明
```

#### 2.3 范围界定的核心原则

列出2-4条原则，每条包含"原则名称"和"具体说明"

### 三、总体业务流程概览

写作要求：
- 本章从全局视角描述核心业务流程框架
- 每个流程按统一结构：触发起点 → 核心环节 → 流程终点
- 最后用专门一节描述各流程之间的交互关系和数据闭环

每个流程模板：
```
### 流程N：XXX流程

描述句：说明该流程的定位和目标。

触发起点：
- 方式一：XXX
- 方式二：XXX

核心环节：
- 环节一：XXX
- 环节二：XXX

流程终点：XXX
```

闭环关系模板：
```
### 核心流程交互与数据闭环

描述各流程之间不是孤立的，而是通过XXX形成闭环：
- 流程A为流程B提供XXX
- 流程B的结果反哺流程A的XXX
- 流程C扩展了XXX的能力

通过上述协同运作，系统实现了从XXX到XXX的完整业务闭环。
```

### 四、系统功能（核心章节）

每个功能模块/专题按以下子结构展开。

**标准结构**（适用于4+页面的复杂模块）：N.1业务目标 → N.2业务场景 → N.3业务流程 → N.4功能模块详情

**简化结构**（适用于1-3个页面的简单模块）：N.1业务场景与流程（合并） → N.2功能模块详情

根据模块复杂度选择合适的结构层级。

#### N.1 业务目标（标准结构）

3-5条，每条一句话，使用动宾结构

```
- 目标一：动词 + 具体效果
- 目标二：动词 + 具体效果
```

#### N.2 业务场景

2-4个场景，每个场景用"角色+情境+动作"格式

```
场景一（场景名称）：某角色在某情况下需要做某事。他可以通过XXX方式，完成XXX操作。
场景二（场景名称）：某角色在某情况下需要做某事...
```

#### N.3 业务流程（标准结构）

文字描述流程，配合列表展示关键步骤和状态流转

```
流程描述：
- 步骤一：XXX
- 步骤二：XXX
- 步骤三：XXX
```

#### N.4 流程环节/功能模块

每个页面/操作严格使用以下模板：

---

**A类（传统功能）页面/操作模板：**

```markdown
##### N.4.X 页面/操作名称

【操作入口】：描述从哪里进入（菜单路径或父页面按钮）

【功能描述】：一句话概述该功能解决什么问题

【使用角色】：列出使用该功能的角色

【界面原型】：
查询区域（顶部）：
- 条件1、条件2...
- 按钮：【查询】【重置】【新增】

列表区域（中部）：
- 表格列：字段1、字段2...
- 操作列：【按钮1】【按钮2】...

分页区域（底部）：
- 支持每页10/20/50/100条

【实现逻辑】：
页面初始化：
- 进入页面时的默认行为

按钮逻辑：
- 【按钮1】逻辑：
  - 权限：谁可操作
  - 操作：点击后的具体行为
  - 校验：需要做哪些验证
  - 结果：操作成功后的效果

状态流转：
- 从状态A → 触发条件 → 状态B

【字段说明】：

| 字段名 | 类型 | 必填 | 可编辑 | 说明 |
|--------|------|------|--------|------|
| 字段1 | 文本 | 是 | 只读 | 说明 |
| 字段2 | 下拉 | 是 | 可选择 | 枚举值：A/B/C |
```

**B类（AI/异步/批处理功能）页面/操作模板：**

B类指需要异步处理、长时运行、AI推理或批量操作的功能。在A类模板基础上，按实际情况选用以下要素：

```markdown
【交互过程】：（AI推理或长时异步任务必填）
步骤1：正在XXX...
步骤2：正在XXX...
步骤3：XXX完成

【提示词(Prompt)】：（仅AI功能需要）
# Role: XXX专家
# Profile: 擅长XXX
# Constrains:
1. 约束1
2. 约束2
# Workflow:
1. 步骤1
2. 步骤2
# OutputFormat: JSON/文本/表格

【语料/知识库】：（仅AI功能需要）
1. 数据源1：XXX
2. 数据源2：XXX

【异步/批处理机制】：（仅异步/批处理功能需要）
- 触发条件：XXX
- 执行策略：定时/事件驱动/手动触发
- 并发控制：XXX
- 结果通知：消息推送/页面轮询/回调

【验证效果】：（B类功能均需填写）
- AI功能：准确率/召回率等量化指标
- 批处理功能：成功率/吞吐量/平均处理时间
- 异步功能：完成率/超时率
```

### 五、非功能需求

#### 5.1 性能要求

##### 5.1.1 页面性能

| 大类 | 小类 | 类型 | 指标项 | 要求描述 |
|------|------|------|--------|----------|
| 页面性能 | 首屏加载 | A类 | 平均响应时间 | 核心页面首屏渲染 ≤2秒 |
| 页面性能 | 流式/进度交互 | B类 | 状态响应连贯性 | 异步处理状态切换延迟 ≤300ms |

##### 5.1.2 功能性能

| 大类 | 小类 | 类型 | 指标项 | 要求描述 |
|------|------|------|--------|----------|
| 功能性能 | 事务处理 | A类 | 数据操作响应 | 核心事务操作响应 ≤3秒 |
| 功能性能 | AI推理 | B类 | 端到端处理时间 | AI处理完成 ≤N秒 |
| 功能性能 | 批处理 | B类 | 吞吐量 | 每批次处理 ≥N条/分钟 |
| 功能性能 | 异步任务 | B类 | 完成时限 | N分钟内完成 |

根据项目实际情况选择适用的行，删除不涉及的B类行。纯A类项目可删除所有B类行。

##### 5.1.3 接口性能

| 大类 | 小类 | 类型 | 指标项 | 要求描述 |
|------|------|------|--------|----------|
| 接口性能 | 同步接口 | A类 | 响应时间/TPS | 95%请求 ≤500ms |
| 接口性能 | 异步接口 | B类 | 超时控制 | 超时N秒返回错误或转入队列 |

#### 5.2 信息安全

##### 5.2.1 权限控制
- 菜单与功能权限：描述按钮级别的权限控制
- 数据权限：基于角色的数据可见范围

##### 5.2.2 展示加密
描述敏感信息在前端的脱敏展示规则

##### 5.2.3 存储加密
描述敏感字段在数据库中的加密方式（如AES-256）

##### 5.2.4 传输加密
描述通信协议要求（如HTTPS/TLS 1.2+）及认证鉴权机制
